## レビュー範囲（重要）

レビュー対象は、Pull Request に含まれるすべての変更ファイルです。

レビューは変更行（diff）のみでなく、以下も必ず確認してください：

- 変更ファイル内の未変更部分も含めて、構造・class名・整合性に問題がないか確認する
- 変更に関連するHTML / CSS / JavaScript間の整合性を確認する
- 変更により既存レイアウトや既存機能が破綻していないか確認する

変更行に直接含まれていない箇所でも、関連する問題を発見した場合は必ず指摘してください。

## レビューの優先順位（重要度順）

### Must-fix（必須）
以下は必ず指摘してください：

- レイアウト崩れの原因となる問題
- class名の不一致
- 画像・CSS・JSのパスミス
- JavaScriptエラーの可能性
- null参照の可能性
- HTML構造の破綻
- FLOCSS/BEM構造の破綻
- specificity過剰（IDセレクタ、過剰ネストなど）

### Nice-to-have（改善提案）
以下は必要な場合のみ指摘してください：

- 保守性の低下につながる書き方
- 不要な重複コード
- 可読性が著しく低いコード

## このプロジェクトの前提

- 静的LPサイト（HTML / CSS / JavaScript）
- **mobile-firstで実装する**
- **デザインカンプはpx基準**
- 実装はカンプの再現を最優先とする

重要：

- px指定は正しい実装であり、remへの変換は提案しないこと
- 色・余白・フォントサイズの値自体の適否は指摘しないこと（デザイン仕様のため）

---

## HTMLレビュー規則（実装ミス検出）
以下を確認してください：

- 見出し構造が正しい（h1→h2→h3の順序）
- class名がCSSと一致している
- imgに適切なaltがある（装飾画像はalt=""）
- リンク・ボタン等インタラクティブ要素の入れ子がない
- 明らかに不要なdivネストがない

加えて、実装ミスとして以下も必ず確認：
- 閉じタグ漏れ・入れ子の破綻がない（HTML構文エラーの疑い）
- idの重複がない（同一idが複数存在しない）
- href/srcなど主要属性の書き間違い・未設定がない
- href="#" が本番に残っていない（仮リンクは明示コメントがある場合のみ許容）
- フォームがある場合：labelとinputが正しく紐づいている（for/id）

指摘対象外：

- 色
- フォント
- 余白量
- デザインの見た目の判断

## CSSレビュー規則（FLOCSS + BEM）

### 命名規則

FLOCSSレイヤーを守る：

- l-（Layout）
- p-（Project）
- c-（Component）
- u-（Utility）
- is-（State）
- js-（JavaScript hooks）

BEM構造：

block__element--modifier

例：

p-card__title  
p-button--primary  

---

### 設計規則

必ず指摘する：

- IDセレクタによるスタイリング
- 過剰なネスト（3階層以上など）
- !importantの使用（理由がない場合）

指摘しない：

- px使用
- rem未使用
- デザイン値そのもの

---

### レスポンシブ規則（mobile-first）

正しい：

```css
.block {
  font-size: 14px;
}

@media (min-width: 768px) {
  .block {
    font-size: 16px;
  }
}
```
